检查本机是否存在golang解释器

步骤 1：设置 GOROOT（ Go 环境所在目录的配置）

为了使所有的计算机用户都可以使用 Go 环境，我们就在系统变量之中配置。点击系统变量下的新建，在变量名一栏输入 GOROOT ，在变量值一栏输入 你解压文件所在的目录D:\路径\go。
最后点击确定，就将 GOROOT 新建完毕。

  1. 右键「此电脑」→ 选择「属性」→ 点击「高级系统设置」→ 点击「环境变量」。
  2. 在「系统变量」中点击「新建」。
  3. 变量名：GOROOT
  4. 变量值：Go 的安装目录（如 C:\Program Files\Go 或 D:\Go）。

步骤 2：设置 GOPATH（工作区路径，存放项目和依赖）

GOPATH和GOROOT的配置略有不同，我建议配置两个GOPATH目录，第一个用于放 Go 语言的第三方包，第二个用于放自己的开发代码。我们来新建GOPATH。点击系统变量下的新建，在变量名一栏输入GOPATH，在变量值一栏输入任意两个目录，中间用英文分号隔开。

示例：D:\路径\go\library;D:\路径\go\workspace

  1. 在「系统变量」中点击「新建」（若不存在则新建）。
  2. 变量名：GOPATH
  3. 变量值：自定义工作区路径（如 D:\路径\go\library;D:\路径\go\workspace，建议单独创建空目录）。
    • 说明：Go 1.13 后支持模块（go mod），GOPATH 的作用弱化，但仍用于存放 go install 安装的工具（如 gopls 语言服务器）。

步骤 3：设置 Path（添加 Go 命令路径）

  1. 在「系统变量」中找到 Path 变量，双击打开编辑界面。
  2. 点击「新建」，添加以下路径：
    • %GOROOT%\bin（Go 编译器、工具（如 go、gofmt）的路径）。

校验是否配置成功

windows+R 输入 cmd 打开终端，输入go version，如果输出如下图所示，则安装成功。

配置 GO111MODULE、GOPROXY、GOSUMDB

Go默认的GOPROXY的值是：GOPROXY=https://proxy.golang.org,direct。这个goproxy在使用go get安装第三方库的时候会报错，导致无法下载成功，所以必须要修改一下。

比如改为：https://goproxy.io,direct （七牛镜像）或 https://mirrors.aliyun.com/goproxy（阿里云镜像）

java 解释器配置

1、检查本地是否配置java jdk环境

不存在则配置环境变量

2、下载jdk11版本

3、window系统环境变量配置

步骤 1：打开环境变量设置界面

  • Windows 11/10：右键「此电脑」→ 选择「属性」→ 点击左侧「高级系统设置」→ 点击右下角「环境变量」。

步骤 2：设置 JAVA_HOME（必选）

  1. 在「系统变量」区域点击「新建」。
  2. 变量名：JAVA_HOME（严格区分大小写）。
  3. 变量值：JDK 的安装目录（如 C:\Program Files\Java\jdk-11.0.20，需根据实际安装路径调整）。
    • 验证：确保路径末尾是 jdk-11.x.x（非 jre 目录），且包含 bin、lib 子目录。

步骤 3：设置 Path（必选）

  1. 在「系统变量」中找到 Path 变量，双击打开编辑界面。
  2. 点击「新建」，添加以下两个路径（按顺序）：
    • %JAVA_HOME%\bin（JDK 命令工具路径，如 java、javac）。
    • %JAVA_HOME%\jre\bin（可选，若需兼容旧版 JRE 相关工具）。
    • 注意：Windows 10/11 支持直接输入变量名，系统会自动解析为 C:\Program Files\Java\jdk-11.0.20\bin。

步骤 4：设置 CLASSPATH（可选，仅旧项目需要）

  1. 在「系统变量」中点击「新建」（若不存在则新建）。

  2. 变量名：CLASSPATH。

  3. 变量值：

  • 说明：. 表示当前目录（运行类文件时需依赖），dt.jar 和 tools.jar 是 JDK 核心库（旧版项目可能需要）。

4、验证是否配置成功
