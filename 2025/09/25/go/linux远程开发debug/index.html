golang 系统 远程环境调试

golang远程linux环境开发调试

1、linux环境配置go环境(最好和本地相同)

go_1.20 Sdk下载地址

下载sdk 此处使用1.20版本 ,将下载好的tar.gz 上传到linux服务器 /usr/local 上面 ,然后解压

创建GOPATH目录(相当于workplace)

编辑配置环境变量

验证是否安装成功

2、goland 配置ssh session 连接会话

goland-file-setting 里面搜索 SSH Configurations 配置 好ssh服务器信息 进行保存

goland-tools-start ssh session 选择配置的ssh服务器连接

3、goland 配置远程 执行

右上角run的位置 下拉 Edit Configurations

点击加号 选择 go build 然后 Run on选择 ssh 选到前面配置的ssh连接 等待检验

RUN kind 选择 Directory

Directory 选择 本地目录

Working directory 选择本地目录

然后通过配置好的 go build 进行debug 或者run即可

4、goland配置文件映射

4.1 配置远程开发机

ctl+alt+s（设置） - 搜索Deployment - Configuration - ➕ - SFTP - 配置主机IP、端口、登录用户、鉴权方式等

选择远程主机配置 ，Root path 可以选择不填 也可以自动填充，相当于mapping里面的前缀

创建完成应用即可 ，映射是针对目录级别的

4.2 查看远程开发配置

Tools - Deployment - Browse Remote Host

此处由于文件过多 更改成了java目录

绿色的就是映射的目录了

4.3 第一次同步需要手动

Tools - Deployment - Sync with local - Synchronize All

4.4 勾选自动同步

Tools - Deployment - Automatic Upload (Always)
这时候在Goland的修改都会自动同步到远程开发机了

4.5 至此配置完成
