Here-Document（heredoc）写入文件

Here-Document（简称 “heredoc”）是 Linux Shell 中一种处理多行文本输入的语法，通过定义 “定界符”（Delimiter）来标记文本的开始和结束，无需手动添加换行符（\n）或转义特殊字符（如引号、$、* 等），能完整保留文本的格式和语义

关键规则

  1. 第一行的 << 定界符 表示 “开始接收多行输入，直到遇到单独一行的定界符为止”。
  2. 结尾的定界符必须单独占一行，且前后不能有任何字符（包括空格、Tab），否则会失效。
  3. 定界符可自定义（如 EOF、TEXT、SCRIPT 等，通常用 EOF 作为默认值，即 “End Of File” 的缩写）

heredoc 的核心灵活性来自定界符是否加引号，这直接决定了文本中 “变量是否解析”“特殊字符是否转义”，分为两种模式：

定界符格式 变量解析 特殊字符处理（如 $、"、\） 本质
无引号（如 EOF） 会解析（替换为实际值） 需手动转义（否则按 Shell 语法解析） 类似 echo "双引号内容"
带引号（如 "EOF"、'EOF'、\EOF） 不解析（原样保留） 无需转义（全部视为纯文本） 类似 echo '单引号内容'

17 linux环境变量等级

在Linux环境中，存在不同的环境变量等级，它们决定了变量的作用范围和优先级

  1. 全局系统级别（Global System Level）：
    这是最高级别的环境变量，适用于整个系统的所有用户和进程。这些环境变量通常定义在系统范围的配置文件中，例如/etc/profile、/etc/environment和/etc/profile.d/目录下的文件。修改这些文件需要root权限。
  2. 用户级别（User Level）：
    用户级别的环境变量仅适用于特定用户。每个用户都有自己的环境变量配置，通常保存在用户的主目录中的文件中。最常见的是~/.bashrc和~/.bash_profile文件，它们在用户登录时会被加载。修改这些文件只会影响到相应用户。
  3. 会话级别（Session Level）：
    会话级别的环境变量仅适用于当前会话。当用户登录后，会话级别的环境变量会被创建，并在用户退出登录后被销毁。这些环境变量可以通过在终端中直接设置变量来定义，例如使用export命令。
  4. 会话级别（Session Level）：
    会话级别的环境变量仅适用于当前会话。当用户登录后，会话级别的环境变量会被创建，并在用户退出登录后被销毁。这些环境变量可以通过在终端中直接设置变量来定义，例如使用export命令。
